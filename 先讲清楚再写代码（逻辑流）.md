#  一、系统设计（先讲清楚再写代码）

## 🔹 1. 小逻辑流（核心）

```
用户输入任务
   ↓
Planner（任务拆解 + 推理）
   ↓
任务队列（状态管理）
   ↓
Executor（执行）
   ↓
Reviewer（检查）
   ↓
Decision：
   ├── 通过 → 写入Memory → 结束
   └── 不通过 → 回到Executor（重试）
```

👉 这就是你要的**“小逻辑流闭环”**

------

## 🔹 2. 是否包含“长链推理（CoT）”

✔ 有，但要控制方式：

- Planner 内部使用：
  - **思考链（Chain-of-Thought）**
  - 任务分解
- 但：
  - ❌ 不暴露完整推理（避免冗余）
  - ✅ 只输出结构化结果

示例：

```
思考：
1. 用户要市场报告
2. 需要数据 + 分析 + 输出
3. 拆分为3个子任务

输出：
["收集数据", "分析数据", "生成报告"]
```

------

## 🔹 3. 多Agent协作（真正协作，不是多线程）

| Agent        | 职责        |
| ------------ | ----------- |
| Planner      | 推理 + 拆解 |
| Executor     | 执行        |
| Reviewer     | 判断质量    |
| Orchestrator | 控制流程    |
